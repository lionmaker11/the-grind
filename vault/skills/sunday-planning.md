# Skill: Sunday Planning

## Trigger
Sunday 6:00 PM heartbeat OR T.J. says "Sunday planning" or "weekly review."

## Part 1: Week in Review (Chief presents)
1. Read all `daily-briefs/` from this week.
2. Compile Pomodoro scorecard: completed vs planned.
3. Category breakdown: which of the 7 categories got attention? Which got zero?
4. Project tracker: which active projects were touched? Which were ignored?
5. Financial check: any income logged? Any bills surfaced? Any financial avoidance?
6. Growth vector check: did any of the 6 growth vectors from [[NORTH_STAR]] trigger this week?

## Part 2: Accountability (Chief asks T.J.)
- "What went well this week?"
- "What did you avoid that you know matters?"
- "Any new ideas that came up?" → Log to Backlog.
- "Any projects that need more or less time next week?"

## Part 3: Next Week Planning (Chief proposes)
1. Read [[NORTH_STAR]] for priority stack.
2. Check Google Calendar for next week's commitments.
3. Check Week A/Week B toggle for [[aurelia|Aurelia]]'s schedule.
4. Propose top 5 priorities with project + category tags.
5. Propose category targets for the week.
6. Pre-populate Basecamp tasks from Backlog based on block schedule.

## Part 4: Financial Review (Chief forces this)
- "Let's look at [[FINANCES]]. Anything new to add?"
- Surface any overdue items.
- Log any deals that closed or income received.
- Ask about upcoming expenses.

## Part 5: Vault Maintenance (Chief runs silently)
- Promote patterns from daily logs to [[MEMORY]].
- Prune daily logs older than 30 days.
- Update [[NORTH_STAR]] if T.J. stated any value shifts.
- Archive completed tasks in Basecamp.
- Flag any system improvement proposals.

## Part 6: Close
"Week is planned. Base Camp is loaded. Monday [[morning-brief|morning brief]] hits at 6 AM. Enjoy your Sunday evening."

## Rules
- Use Sonnet for Sunday planning (more strategic reasoning).
- Financial review is NOT optional. Chief does it even if T.J. resists.
- Keep total planning session to 30 minutes max via Telegram.
